# LLaVA (Large Language and Vision Assistant)

## Description

**LLaVA (Large Language and Vision Assistant)** is an open-source multimodal model that combines a vision encoder (usually CLIP or a ViT-based model) with an open-source Large Language Model (LLM) (such as Vicuna or Llama) to enable understanding and reasoning over images and text. Its unique value proposition lies in being a high-performance open-source alternative to GPT-4V(ision), achieving comparable-level visual conversation capabilities through a technique called *Visual Instruction Tuning*. LLaVA is trained on a multimodal dataset generated by GPT-4, which allows it to follow complex instructions involving both visual analysis and natural language generation. The most recent versions, such as LLaVA-1.5 and LLaVA-NeXT, continue to improve efficiency and performance, making it a fundamental tool for the research and development of open-source MLLMs (Multimodal Large Language Models).

## Statistics

**Parameters:** Available in sizes of 7 Billion (7B) and 13 Billion (13B) parameters. **Performance:** The LLaVA-1.5 version achieved state-of-the-art performance on 11 vision-language benchmarks, surpassing previous open-source models and approaching the performance of GPT-4V on several tasks. **Dataset:** Trained on a dataset of 150K visual and language instruction pairs generated by GPT-4. **Speed:** Optimized for inference, with implementations such as vLLM that offer high throughput.

## Features

**Modular Architecture:** Combines a vision encoder (e.g., CLIP ViT-L/14) and an LLM (e.g., Vicuna, Llama 2) connected by a linear projection layer. **Visual Instruction Tuning:** Trained on a high-quality visual instruction dataset generated by GPT-4 to align vision and language capabilities. **High-Performance Capability:** Achieves state-of-the-art results on 11 vision-language benchmarks, including VQA, GQA, and ScienceQA, with 7B and 13B parameter models. **Multi-Resolution Support:** Recent versions (LLaVA-NeXT) support different aspect ratios and image resolutions, improving the ability to process high-resolution images. **Optimized Inference:** Compatible with fast inference frameworks such as vLLM and Ollama, enabling efficient deployment and scalability.

## Use Cases

**Visual Conversation Assistant:** Answering complex questions about the content of an image, such as "What is happening in this scene and why is object X there?". **Document and Image Analysis:** Extraction of information from charts, tables, and scanned documents, enabling business process automation. **Robotics and Computer Vision:** Providing high-level reasoning for robotic systems, helping them understand the visual environment and plan actions. **Captioning and Detailed Descriptions:** Creating rich and contextual descriptions for images, useful for accessibility and content cataloging. **Education and Tutoring:** Explaining visual concepts in textbooks or diagrams based on user questions.

## Integration

LLaVA's integration is facilitated by its open-source nature and compatibility with popular ML ecosystems. **1. Hugging Face Transformers:** The most common method is to use the `transformers` library to load and run the model.
```python
from transformers import LlavaProcessor, LlavaForConditionalGeneration
from PIL import Image

model_id = "llava-hf/llava-1.5-7b-hf"
processor = LlavaProcessor.from_pretrained(model_id)
model = LlavaForConditionalGeneration.from_pretrained(model_id)

# Load image and prompt
image = Image.open("path/to/your/image.jpg")
prompt = "USER: Describe this image in detail. ASSISTANT:"

inputs = processor(text=prompt, images=image, return_tensors="pt")

# Generate response
output = model.generate(**inputs, max_new_tokens=200)
print(processor.decode(output[0], skip_special_tokens=True))
```
**2. vLLM (OpenAI-Compatible API Service):** vLLM can be used to serve LLaVA with high throughput, exposing an OpenAI-compatible API, which allows integration with tools like LangChain and OpenAI clients.
```bash
# Serve the model with vLLM
python3 -m vllm.entrypoints.openai.api_server --model llava-hf/llava-1.5-7b-hf
```
**3. Ollama:** For easy local use, Ollama offers a simple way to download and run LLaVA.
```bash
# Download and run with Ollama
ollama run llava
```
