# Distributed Training Projects

This directory covers project examples that train deep-learning models across multiple GPUs, nodes, or accelerators.

## What Belongs Here

- end-to-end examples with problem statement, data, model, evaluation, deployment context, and owner
- source links, reproducibility commands, environment notes, and validation evidence
- known failure cases, ethical constraints, cost notes, and next maintenance action

## Source Links

- [PyTorch documentation](https://pytorch.org/docs/stable/index.html)
- [TensorFlow guide](https://www.tensorflow.org/guide)
- [Keras API](https://keras.io/api/)
- [Hugging Face Transformers](https://huggingface.co/docs/transformers/en/index)
- [Papers with Code methods](https://paperswithcode.com/methods)

## Evidence To Track

- Source URL, publication or update date, license, owner, and access conditions.
- Dataset/model/prompt version, benchmark split, metric, baseline, and known limitations when applicable.
- Clear separation between primary evidence, reproduced results, and AIForge interpretation.

## Routing Rules

- Move reusable theory, models, datasets, and prompts back to their source directories.
- Keep showcase files evidence-backed and reproducible, not just inspirational.
- Use domain vertical folders when a project becomes a mature applied workflow.

## Next Enrichment Tasks

- Add high-authority papers, official docs, benchmark entries, and reproducible examples for this topic.
- Add local examples only when they include provenance, validation notes, and maintenance owner.
- Cross-link mature items to the relevant model, dataset, MLOps, or vertical-application directory.
