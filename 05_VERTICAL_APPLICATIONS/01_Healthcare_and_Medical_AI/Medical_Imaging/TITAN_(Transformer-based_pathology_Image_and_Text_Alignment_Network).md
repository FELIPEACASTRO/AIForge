# TITAN (Transformer-based pathology Image and Text Alignment Network)

## Description

TITAN (Transformer-based pathology Image and Text Alignment Network) is a state-of-the-art multimodal foundation model for digital pathology. It was pre-trained using 335,645 whole-slide images (WSIs) and vision-language alignment with corresponding pathology reports, plus 423,122 synthetic captions generated by a multimodal generative AI copilot. Although it uses a Vision Transformer (ViT) architecture, it builds on features extracted by models such as CONCHv1.5, which are essential for large-scale pathology image analysis. TITAN produces general-purpose slide representations and can generate pathology reports, generalizing to resource-limited clinical scenarios such as rare-disease retrieval and cancer prognosis.

## Statistics

- **Pre-training Database**: 335,645 Whole-Slide Images (WSIs) and 182,862 medical reports.
- **Performance (Subtyping)**: Outperforms previous foundation models (such as PRISM, GigaPath, and CHIEF) with an average improvement of **+8.4%** on multiclass subtyping tasks and **+6.7%** on binary tasks.
- **Citations**: 79 citations (as of November 2025, according to Nature Medicine).
- **Publication**: *Nature Medicine* (2025).
- **Rare Cancer Retrieval**: Demonstrates superior performance in precision@K and MVAcc@K on rare cancer datasets.

## Features

- **Multimodality**: Aligns visual representations of WSIs with morphological descriptions and clinical reports (text).
- **General-Purpose Representation**: Extracts slide *embeddings* that can be deployed across diverse clinical settings without the need for extensive *fine-tuning*.
- **Long-Context Modeling**: Uses ALiBi for long-context modeling, enabling accurate predictions across whole-tissue contexts.
- **Report Generation**: Ability to generate high-quality pathology reports, capturing tissue site, diagnosis, and tumor grade.
- **Architecture**: Based on a Vision Transformer (ViT) with student-teacher knowledge distillation, pre-trained in three stages (unimodal vision, ROI-level cross-modal alignment, and WSI-level cross-modal alignment).

## Use Cases

- **Cancer Classification and Subtyping**: Identification of morphological subtypes and molecular classification in WSIs.
- **Biomarker Prediction**: Prediction of molecular biomarkers directly from histopathology images.
- **Rare Case Retrieval**: Retrieval of clinically relevant and morphologically similar cases for rare or complex diseases.
- **Cancer Prognosis**: Assistance in predicting clinical outcomes and survival.
- **Report Generation**: Automated generation of high-quality pathology reports.

## Integration

Integration is done through the model's Python library, which allows the extraction of slide *embeddings* from pre-extracted *patch* features (for example, using CONCHv1.5 or CLAM).

**Example of Slide *Embedding* Extraction (Python/PyTorch):**

```python
import torch
import h5py
from huggingface_hub import hf_hub_download

# The TITAN model must be loaded beforehand
# model = ... 
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = model.to(device)

# Load sample data (pre-extracted patch features)
demo_h5_path = hf_hub_download(
    "MahmoodLab/TITAN", 
    filename="TCGA_demo_features/TCGA-PC-A5DK-01Z-00-DX1.C2D3BC09-411F-46CF-811B-FDBA7C2A295B.h5",
)
file = h5py.File(demo_h5_path, 'r')
features = torch.from_numpy(file['features'][:])
coords = torch.from_numpy(file['coords'][:])
patch_size_lv0 = file['coords'].attrs['patch_size_level0']

# Extract the slide embedding
with torch.autocast('cuda', torch.float16), torch.inference_mode():
    features = features.to(device)
    coords = coords.to(device)
    slide_embedding = model.encode_slide_from_patch_features(features, coords, patch_size_lv0)

# 'slide_embedding' contains the vector representation of the whole slide.
```

The project also recommends using **TRIDENT** for large-scale feature extraction. Access to the model weights is provided through MahmoodLab's Hugging Face page.
