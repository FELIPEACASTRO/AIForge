# Awesome-List Submissions — AIForge (ready-to-PR)

Getting AIForge into well-known "awesome" lists is one of the highest-ROI discoverability moves: each accepted PR is an authoritative backlink + a notability source (which also unlocks the Wikidata item).

```

```

## PR checklist (per list)
1. **Read** the list's `contributing.md` / `code-of-conduct`.
2. **Fork** → add the entry in the **correct section**, alphabetical if required.
3. Match **punctuation & description length** exactly.
4. Ensure the AIForge repo itself: has a clear description, topics, a license, and (ideally) the **Awesome badge** + passing **awesome-lint** (a CI workflow is already in `.github/workflows/awesome-lint.yml`).
5. **PR title:** `Add AIForge` · **PR body:** one sentence on what it is + why it fits the list.
6. Be patient & responsive to maintainer feedback.

## Order of operations (maximizes acceptance)
1. Make the repo **public**, polish description + topics, enable Pages.
2. Get a little initial traction (share via PROMOTION_KIT.md; a few stars help).
3. Submit to 2-3 most-relevant lists first (machine-learning, deep-learning, LLM).
4. Use an accepted PR as the **Wikidata notability reference** (see WIKIDATA_ITEM.md).
