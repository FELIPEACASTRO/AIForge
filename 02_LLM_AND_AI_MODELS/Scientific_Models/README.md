# Scientific Models

This directory covers AI models built for scientific discovery, simulation, forecasting, and domain-specific reasoning. It should connect model cards and papers to datasets, benchmarks, physical constraints, and reproducibility notes.

## Subdirectories

| Subdirectory | Scope |
|---|---|
| `Protein_Structure/` | AlphaFold-style structure prediction, protein design, molecular representation learning, and structural-biology models. |
| `Weather_Climate_Models/` | Weather forecasting, climate emulation, earth-system models, nowcasting, and geospatial foundation models. |
| `Quantum_ML/` | Quantum machine learning, quantum circuits, hybrid classical-quantum learning, and quantum-inspired models. |
| `Reservoir_Computing/` | Echo state networks, liquid state machines, stochastic reservoirs, and dynamical-system forecasting. |

## Enrichment Standard

Scientific-model pages should include task definition, input/output format, datasets, benchmark metrics, domain assumptions, uncertainty or calibration notes, and whether the model is suitable for research, production, or decision support.

## Routing Rules

- Put medical applications in `../../05_VERTICAL_APPLICATIONS/01_Healthcare_and_Medical_AI/`.
- Put climate applications in `../../05_VERTICAL_APPLICATIONS/04_Climate_and_Sustainability/`.
- Put deployment or acceleration in `../../04_MLOPS_AND_PRODUCTION_AI/`.
