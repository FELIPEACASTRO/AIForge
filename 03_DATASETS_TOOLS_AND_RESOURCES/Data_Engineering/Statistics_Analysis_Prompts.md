# Statistics Analysis Prompts

## Description
Statistics Analysis Prompts are structured, detailed instructions provided to Large Language Models (LLMs) to guide the execution of complex data science and statistics tasks. Instead of merely asking for a result, the prompt acts as a 'virtual data analyst', defining the context, the analysis objective, the nature of the data (whether by upload or description), the statistical method to be applied (such as hypothesis testing, regression, or time series analysis), and the desired output format (code, executive summary, visualization). The goal is to mitigate the inherent limitations of LLMs in mathematical reasoning and ensure the correct application of statistical methodologies, transforming raw data into actionable insights.

## Examples
```
1. **Hypothesis Testing (T-Test):** 'Assuming the sales data is in [file/table name], perform a two-sample t-test to determine whether there is a statistically significant difference in average revenue between 'Group A' (customers who used a coupon) and 'Group B' (customers without a coupon). Present the null hypothesis, the p-value, and the conclusion in business terms.'\n\n2. **Regression Analysis:** 'Use the data from [dataset name] to build a multiple linear regression model to predict the variable 'customer satisfaction' based on the variables 'support response time', 'product price', and 'purchase frequency'. Provide the Python/R code for the model, the adjusted R², and the interpretation of the coefficients.'\n\n3. **Exploratory Data Analysis (EDA):** 'Perform a complete Exploratory Data Analysis (EDA) on the dataset [dataset name]. Include descriptive statistics (mean, median, standard deviation), identify and visualize the distribution of the key variables, and point out any anomalies or missing values. Summarize the top 3 insights found.'\n\n4. **Time Series Analysis:** 'For the time series of [metric name, e.g., website traffic] over the period from [start date] to [end date], identify the trend and seasonality. Apply an ARIMA or Prophet model to forecast the next 6 months. Describe the chosen model and the forecast accuracy.'\n\n5. **Data Cleaning and Preprocessing:** 'The dataset [dataset name] contains missing values marked as 'NA' and outliers in the 'age' column (above 100). Write a script that handles the missing values with the median and removes the outliers. Then, normalize the 'income' column using Z-score standardization.'\n\n6. **Customer Segmentation (Clustering):** 'Using the columns 'total_amount_spent', 'purchase_frequency', and 'days_since_last_purchase', apply the K-Means algorithm to segment the customers into 4 distinct groups. Describe the characteristics of each cluster and suggest a name for each segment (e.g., 'VIP Customers').'\n\n7. **Results Interpretation:** 'The result of the hypothesis test for marketing campaign A vs B returned a p-value of 0.03. Explain what this value means for a non-technical executive audience and provide a clear action recommendation based on this result.'
```

## Best Practices
1. **Define the Role and Context:** Begin the prompt by defining the LLM as a 'Statistics Expert' or 'Data Scientist' and provide the context of the analysis (e.g., 'You are a marketing analyst evaluating the performance of a campaign').\n2. **Coherent Structure:** Use the structure 'Context -> Data -> Objective -> Method -> Output Format'. Be explicit about which statistical test or model should be used.\n3. **Data Specificity:** If the data cannot be loaded, describe the structure (columns, data types, number of rows) and the problem (e.g., 'The 'revenue' column is numeric and contains some null values').\n4. **Validation and Interpretation:** Ask the LLM not only to generate the code or result, but also to interpret the statistical meaning and practical relevance of the findings, acting as a quality filter.\n5. **Iteration and Refinement:** Use follow-up prompts to refine the analysis (e.g., 'Now, explore the correlation between [variable X] and [variable Y] for Cluster 3 that you identified').

## Use Cases
1. **Finance and Economics:** Stock price forecasting (time series), credit risk modeling (logistic regression), and volatility analysis.\n2. **Marketing and Sales:** A/B testing of campaigns, customer segmentation (clustering), and product demand forecasting.\n3. **Healthcare and Research:** Analysis of clinical trials (hypothesis testing), disease risk factor modeling (regression), and genomic data analysis.\n4. **Operations and Logistics:** Inventory optimization, equipment failure prediction (survival analysis), and process bottleneck analysis.\n5. **Human Resources:** Analysis of factors influencing employee turnover (regression), and evaluation of the effectiveness of training programs.

## Pitfalls
1. **Statistical Hallucinations:** The LLM may 'invent' results, p-values, or statistically incorrect conclusions if it is not explicitly instructed to use validated code or method.\n2. **Unsuitability for Large Data Volumes:** LLMs are not data processing tools; they cannot handle large datasets directly. The analysis must be done based on summaries, samples, or metadata.\n3. **Generation of Faulty Code:** The generated code (Python/R) may contain syntax or logic errors, especially if the libraries or data structure are not described accurately.\n4. **Bias and Oversimplification:** The LLM may oversimplify complex relationships or introduce bias into the results if the prompt does not require a multifactorial and critical analysis.\n5. **Confusion between Causation and Correlation:** The model may interpret correlation as causation in its interpretation, unless the prompt requires caution and consideration of confounding variables.
