# Data Quality

This directory covers data quality for AI, ML, analytics, and production systems.

## Scope

- Schema checks, missingness, distribution drift, uniqueness, validity, freshness, referential integrity, outliers, and data contracts.
- Track expectation, severity, owner, validation target, alert, and remediation.

## Reference Links

- Great Expectations: https://docs.greatexpectations.io/
- Soda: https://docs.soda.io/
- TensorFlow Data Validation: https://www.tensorflow.org/tfx/data_validation/get_started
- Deequ: https://github.com/awslabs/deequ

## Routing Rules

- Put live model drift in MLOps observability.
- Put data contracts in data management.
