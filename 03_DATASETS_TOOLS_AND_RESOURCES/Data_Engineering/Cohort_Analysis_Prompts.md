# Cohort Analysis Prompts

## Description
Cohort Analysis Prompts are *prompt engineering* techniques aimed at instructing a Large Language Model (LLM) to perform or assist with **Cohort Analysis**. Cohort Analysis is a behavioral analysis method that tracks the behavior of specific groups of users (cohorts) that share common characteristics or experiences (usually time-based, such as the month of first purchase) over time. The use of prompts allows the LLM to process raw data (such as CSVs or tables) to define cohorts, calculate metrics (retention, *churn*, revenue), and generate visualizations or interpretations, especially useful for users unfamiliar with spreadsheet tools or traditional data analysis [1] [2]. Although the LLM may not be the ideal tool for complex data analysis itself, it acts as a powerful **planner, formatter, and interpreter**, allowing the user to obtain a quick and understandable "first cut" of the analysis [1] [3].

## Examples
```
1. **Cohort Table Generation Prompt (Revenue):**
```
You are a Revenue Analyst. Given the following transaction table (CSV), create a right-aligned Cohort Analysis table. The cohort should be defined by the customer's 'month of first purchase'. The columns should represent the subsequent months of transaction. The value in each cell should be the sum of the 'revenue' generated by that cohort in that month.

[Insert here the CSV table of transaction data: customer_id, purchase_date, revenue]
```

2. **Cohort Table Generation Prompt (Percentage Retention):**
```
Based on the provided usage data (user_id, signup_date, activity_month), generate a Retention Cohort Analysis table. The cohort should be defined by the 'signup month'. The value in each cell should be the percentage of users from the original cohort who remained active in the subsequent month. The signup month should be 100%.
```

3. **Results Interpretation Prompt:**
```
Analyze the cohort retention table I just provided. Identify the best-performing cohort and the worst-performing cohort after 3 months. Explain the possible reasons for this difference and suggest an A/B test hypothesis to improve the retention of the worst-performing cohort.
```

4. **Data Visualization Prompt:**
```
Based on the Revenue Cohort Analysis table you generated, create Python code (using Pandas and Matplotlib) to generate a stacked area chart (Layer Cake Chart) that visualizes each cohort's revenue contribution over time.
```

5. **Behavioral Cohort Definition Prompt:**
```
Define 3 behavioral cohorts for a fitness app based on the following criteria: Cohort A: Users who completed 5 workouts in the first week. Cohort B: Users who opened the app but did not complete any workout. Cohort C: Users who canceled the subscription after the trial period. For each cohort, describe 3 key tracking metrics.
```

6. **Analysis Planning Prompt:**
```
Act as a Product Analytics Planner. I want to run a Cohort Analysis to understand the impact of a new feature ('Dark Mode') launched on March 1, 2024. Create a detailed analysis plan, including: 1. Definition of the cohorts (before and after launch). 2. Metrics to be compared. 3. Analysis period. 4. A follow-up prompt for the LLM to execute the analysis.
```

7. **Churn Analysis Prompt:**
```
Using the provided retention cohort table, calculate the *churn* rate for each cohort in the 4th month. Which cohort has the highest and lowest *churn*? Formulate a research question to understand the root cause of the highest *churn*.

## Best Practices
**1. Clear Prompt Structure:** Clearly define the LLM's role (e.g., "You are a Data Analyst"), the data input format (e.g., "The following CSV table"), the cohort definition (e.g., "Cohort based on the month of first purchase"), and the desired output metric (e.g., "Month-by-month retention table"). **2. Provide Context and Goals:** Include the objective of the analysis (e.g., "Identify the cohort with the highest retention after 6 months") to guide the LLM's interpretation. **3. Iteration and Refinement:** Use the LLM to refine the analysis. First ask for the cohort table, then use a follow-up prompt to request the interpretation or visualization (e.g., "Generate a stacked area chart based on the table above"). **4. Human Validation:** Always verify the plausibility of the results generated by the LLM, as they are based on a stochastic model and do not replace data auditing [1].

## Use Cases
**1. Customer Retention Analysis (SaaS):** Track the retention of users who signed up in different months to identify seasonal trends or the impact of product changes. **2. Marketing Campaign Evaluation:** Compare the *Lifetime Value* (LTV) of cohorts of customers acquired through different marketing channels (e.g., Google Ads vs. Social Media). **3. Product Optimization:** Analyze the behavior of cohorts of users who experienced a new feature to measure its impact on retention and engagement. **4. Revenue Analysis:** Create "layer cake" charts (*Layer Cake Charts*) to visualize how older and newer cohorts contribute to total revenue over time [1]. **5. Behavioral Segmentation:** Identify groups of users with similar usage patterns (behavioral cohorts) to target specific communications or offers.

## Pitfalls
**1. Excessive Reliance on Raw Data:** The LLM can only analyze the data provided to it. If the input data (CSV, table) is dirty, inconsistent, or incomplete, the cohort analysis will be flawed. **2. Context Limitations:** For large datasets, the LLM may reach the token limit, resulting in truncation or incomplete analysis. **3. Lack of Plausibility:** The LLM may generate results that appear correct but are statistically implausible or incorrect. Human validation is crucial [1]. **4. Visualization Complexity:** Although LLMs can generate code for visualizations, the execution and refinement of complex charts (such as stacked area charts) may require several prompt iterations and are often more efficient in dedicated tools [1]. **5. Ambiguity in Cohort Definition:** Prompts that do not clearly define the cohort event (e.g., "first login" vs. "first purchase") will lead to inconsistent results.
