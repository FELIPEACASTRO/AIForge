## Coverage Status

| Status | Meaning |
|---|---|
| S0 seeded | Country/entity is present in the matrix; no AI-specific local source has been validated yet. |
| S1 global-index covered | Country appears in at least one credible global AI/readiness/policy index. |
| S2 official national source | Official national AI strategy, ministry page, regulator page, or government AI program added. |
| S3 ecosystem sources | Academic, industry, dataset, startup, compute, and community sources added. |
| S4 verified current | Links checked, dates captured, and claims reconciled against at least two independent sources. |

Current matrix seed after the 2026-07-07 source passes: 195 countries, 127 with `S2 official national source`, 1 with `S1 global-index covered`, and 67 still at `S0 seeded`.

## Source Rules

- Do not claim country coverage unless a source row names the publisher, URL, date or access date, scope, and why it matters.
- Prefer official sources first: government, intergovernmental organizations, standards bodies, research institutions, and primary conference/journal pages.
- Separate global-index evidence from country-specific evidence. A country appearing in Oxford Insights, IMF, OECD, UNESCO, or Stanford data is not the same as having a dedicated national AI page.
- When adding a new country-specific source, update the matrix status and add the source to the most specific existing AIForge section.
- If a source is stale, paywalled, marketing-only, or secondary, label it honestly instead of inflating coverage.

## Expansion Loop

1. Pick a source family: policy, research, dataset, model, benchmark, patent, compute, education, safety, or country ecosystem.
2. Search primary sources first, then cross-check with global indexes.
3. Add only source-backed entries.
4. Update the country matrix and benchmark evidence.
5. Re-run link and scale checks before claiming coverage.
