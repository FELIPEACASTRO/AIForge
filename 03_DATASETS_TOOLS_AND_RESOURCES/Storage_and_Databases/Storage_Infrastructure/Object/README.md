# Object Storage

This directory covers object storage for datasets, model artifacts, logs, checkpoints, and AI assets.

## Scope

- S3-compatible storage, cloud object stores, lifecycle policies, versioning, access control, encryption, multipart uploads, and dataset layouts.
- Track bucket/container, prefix convention, retention policy, object format, access policy, and reproducibility.

## Reference Links

- Amazon S3: https://docs.aws.amazon.com/s3/
- Google Cloud Storage: https://cloud.google.com/storage/docs
- Azure Blob Storage: https://learn.microsoft.com/en-us/azure/storage/blobs/
- MinIO: https://min.io/docs/minio/linux/index.html

## Routing Rules

- Put database engines in database directories.
- Put pipeline orchestration in data-engineering pipeline folders.
