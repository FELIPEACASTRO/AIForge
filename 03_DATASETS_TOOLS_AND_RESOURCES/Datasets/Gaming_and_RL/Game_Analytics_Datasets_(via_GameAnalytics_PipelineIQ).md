# Game Analytics Datasets (via GameAnalytics PipelineIQ)

## Description
The term "Game Analytics Datasets" refers primarily to the raw game event data collected and made available by the GameAnalytics platform, through its **PipelineIQ** product (specifically the **Data Export** feature). These datasets are not a static, public dataset, but rather a real-time and historical data stream that game developers can export to their own cloud environments (such as AWS S3, Google Cloud Storage, or Azure Blob Storage) for in-depth analysis and Machine Learning model training [1].

The exported data includes all gameplay events, design events, business events, ad events, and custom dimensions that the developer tracks in their game. GameAnalytics acts as a data pipeline, allowing game studios to maintain full ownership and control over their raw data for custom and long-term analyses [2].

The platform also offers the **Data Warehouse**, which contains two main datasets for PipelineIQ customers: **Checkpoints Data Sets** (daily aggregated data with retention of up to 1 year) and **Event Tables** (raw event data with retention of up to 30 days), which can be queried directly in BigQuery [3].

## Statistics
The datasets do not have a fixed size or sample count, as they are generated continuously by the developer's game.

*   **Size:** Variable, depending on the volume of events generated by the game (millions to billions of events per day).
*   **Retention (Data Export):** Unlimited, as the data is stored in the client's cloud infrastructure, subject to their internal retention policies.
*   **Retention (Data Warehouse - BigQuery):**
    *   **Checkpoints Data Sets:** Up to 1 year of daily aggregated data.
    *   **Event Tables:** Up to 30 days of raw event data [3].
*   **Versions:** The data reflects the current version of the game and the event structure defined by the developer. GameAnalytics ensures that all custom fields are retained in the data stream [2].

## Features
The main features of GameAnalytics' Game Analytics datasets include:

*   **Real-Time Raw Data:** Export of all gameplay events in real time (streaming) to the client's cloud environment.
*   **Full Flexibility:** There are no rigid or predefined schemas. The data includes all custom fields, events, and dimensions defined by the developer.
*   **Detailed Events:** Capture of design events (player actions), business events (transactions), resource events (game economy), ad events (impressions and clicks), and error events [3].
*   **Ownership and Control:** The developer maintains full ownership of the data, storing it in their own cloud infrastructure for long-term analysis.
*   **BI Compatibility:** Ease of integration with Business Intelligence (BI) tools such as Tableau, Power BI, or Looker [2].

## Use Cases
Game Analytics datasets are crucial for advanced data applications in games:

*   **Custom Business Intelligence (BI):** Creation of tailored BI dashboards and reports that go beyond the standard analyses of the GameAnalytics platform [2].
*   **Player Behavior Analysis:** In-depth analysis of interaction patterns, daily progression, conversion funnels, and identification of drop-off points (churn) [2].
*   **Monetization Optimization:** Detailed analysis of ad engagement and generated revenue to optimize ad placement and frequency [2].
*   **Machine Learning and AI:** Training AI models for churn prediction, player segmentation, offer personalization, and recommendation systems [1].
*   **Long-Term Historical Analysis:** Performing analyses of trends and changes in player behavior over years, without depending on third-party retention policies [2].

## Integration
Integration and use of GameAnalytics' Game Analytics datasets are carried out through the **PipelineIQ** product (**Data Export** feature).

1.  **Subscription:** It is necessary to be a PipelineIQ customer (usually Pro or Enterprise) of GameAnalytics.
2.  **Destination Configuration:** The developer configures a cloud storage destination (AWS S3, Google Cloud Storage, Azure Blob Storage) to receive the data stream.
3.  **Export:** GameAnalytics streams gameplay data in real time to the configured destination, with exports occurring every 15 minutes, provided there is new data [3].
4.  **Analysis:** The raw data is then accessible directly in the client's cloud environment, where it can be processed, transformed, and loaded into data warehouses or used to train Machine Learning models.

For customers using the **Data Warehouse**, access is done via SQL queries in Google BigQuery, where the Checkpoints and Event Tables data are populated [3].
