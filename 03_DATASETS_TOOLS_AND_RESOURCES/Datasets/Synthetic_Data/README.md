# Synthetic Data

This directory covers generated, simulated, augmented, de-identified, or privacy-preserving datasets for AI training, testing, and evaluation.

## What Belongs Here

- dataset catalogs with provenance, owner, license, access method, schema, and update cadence
- task definitions, splits, leakage risks, benchmark metrics, baselines, and known limitations
- preprocessing notes, quality checks, privacy constraints, and reproducible download references

## Source Links

- [Hugging Face Datasets](https://huggingface.co/datasets)
- [Kaggle Datasets](https://www.kaggle.com/datasets)
- [UCI Machine Learning Repository](https://archive.ics.uci.edu/)
- [OpenML](https://www.openml.org/)
- [Google Dataset Search](https://datasetsearch.research.google.com/)

## Evidence To Track

- Source URL, publication or update date, license, owner, and access conditions.
- Dataset/model/prompt version, benchmark split, metric, baseline, and known limitations when applicable.
- Clear separation between primary evidence, reproduced results, and AIForge interpretation.

## Routing Rules

- Put modeling theory in 01_AI_FUNDAMENTALS_AND_THEORY.
- Put named model catalogs in 02_LLM_AND_AI_MODELS.
- Put end-to-end applied examples in 05_VERTICAL_APPLICATIONS.
