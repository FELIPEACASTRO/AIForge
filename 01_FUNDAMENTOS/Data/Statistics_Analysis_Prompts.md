# Statistics Analysis Prompts

## Description
Prompts de Análise Estatística são instruções estruturadas e detalhadas fornecidas a Modelos de Linguagem Grande (LLMs) para guiar a execução de tarefas complexas de ciência de dados e estatística. Em vez de apenas pedir um resultado, o prompt atua como um 'analista de dados virtual', definindo o contexto, o objetivo da análise, a natureza dos dados (seja por upload ou descrição), o método estatístico a ser aplicado (como teste de hipótese, regressão ou análise de séries temporais) e o formato de saída desejado (código, resumo executivo, visualização). O objetivo é mitigar as limitações inerentes dos LLMs em raciocínio matemático e garantir a aplicação correta de metodologias estatísticas, transformando dados brutos em insights acionáveis.

## Examples
```
1. **Teste de Hipótese (T-Test):** 'Assumindo que os dados de vendas estão em [nome do arquivo/tabela], realize um teste t de duas amostras para determinar se há uma diferença estatisticamente significativa na receita média entre o 'Grupo A' (clientes que usaram cupom) e o 'Grupo B' (clientes sem cupom). Apresente a hipótese nula, o valor p e a conclusão em termos de negócio.'\n\n2. **Análise de Regressão:** 'Utilize os dados de [nome do dataset] para construir um modelo de regressão linear múltipla para prever a variável 'satisfação do cliente' com base nas variáveis 'tempo de resposta do suporte', 'preço do produto' e 'frequência de compra'. Forneça o código Python/R para o modelo, o R² ajustado e a interpretação dos coeficientes.'\n\n3. **Análise Exploratória de Dados (EDA):** 'Realize uma Análise Exploratória de Dados (EDA) completa no dataset [nome do dataset]. Inclua estatísticas descritivas (média, mediana, desvio padrão), identifique e visualize a distribuição das variáveis-chave, e aponte quaisquer anomalias ou valores ausentes. Resuma os 3 principais insights encontrados.'\n\n4. **Análise de Séries Temporais:** 'Para a série temporal de [nome da métrica, ex: tráfego do site] no período de [data inicial] a [data final], identifique a tendência e a sazonalidade. Aplique um modelo ARIMA ou Prophet para prever os próximos 6 meses. Descreva o modelo escolhido e a precisão da previsão.'\n\n5. **Limpeza e Pré-processamento de Dados:** 'O dataset [nome do dataset] contém valores ausentes marcados como 'NA' e outliers na coluna 'idade' (acima de 100). Escreva um script que trate os valores ausentes com a mediana e remova os outliers. Em seguida, normalize a coluna 'renda' usando a padronização Z-score.'\n\n6. **Segmentação de Clientes (Clustering):** 'Usando as colunas 'valor_total_gasto', 'frequencia_de_compra' e 'dias_desde_ultima_compra', aplique o algoritmo K-Means para segmentar os clientes em 4 grupos distintos. Descreva as características de cada cluster e sugira um nome para cada segmento (ex: 'Clientes VIP').'\n\n7. **Interpretação de Resultados:** 'O resultado do teste de hipótese para a campanha de marketing A vs B retornou um valor p de 0.03. Explique o que este valor significa para um público executivo não-técnico e forneça uma recomendação de ação clara baseada neste resultado.'
```

## Best Practices
1. **Definir o Papel e o Contexto:** Comece o prompt definindo o LLM como um 'Especialista em Estatística' ou 'Cientista de Dados' e forneça o contexto da análise (ex: 'Você é um analista de marketing avaliando a performance de uma campanha').\n2. **Estrutura Coerente:** Utilize a estrutura 'Contexto -> Dados -> Objetivo -> Método -> Formato de Saída'. Seja explícito sobre qual teste ou modelo estatístico deve ser usado.\n3. **Especificidade dos Dados:** Se os dados não puderem ser carregados, descreva a estrutura (colunas, tipos de dados, número de linhas) e o problema (ex: 'A coluna 'receita' é numérica e contém alguns valores nulos').\n4. **Validação e Interpretação:** Peça ao LLM para não apenas gerar o código ou o resultado, mas também para interpretar o significado estatístico e a relevância prática dos achados, agindo como um filtro de qualidade.\n5. **Iteração e Refinamento:** Use prompts de acompanhamento para refinar a análise (ex: 'Agora, explore a correlação entre [variável X] e [variável Y] para o Cluster 3 que você identificou').

## Use Cases
1. **Finanças e Economia:** Previsão de preços de ações (séries temporais), modelagem de risco de crédito (regressão logística), e análise de volatilidade.\n2. **Marketing e Vendas:** Teste A/B de campanhas, segmentação de clientes (clustering), e previsão de demanda de produtos.\n3. **Saúde e Pesquisa:** Análise de ensaios clínicos (testes de hipótese), modelagem de fatores de risco de doenças (regressão), e análise de dados genômicos.\n4. **Operações e Logística:** Otimização de estoque, previsão de falhas de equipamentos (análise de sobrevivência), e análise de gargalos em processos.\n5. **Recursos Humanos:** Análise de fatores que influenciam a rotatividade de funcionários (regressão), e avaliação da eficácia de programas de treinamento.

## Pitfalls
1. **Alucinações Estatísticas:** O LLM pode 'inventar' resultados, valores p ou conclusões estatisticamente incorretas se não for explicitamente instruído a usar um código ou método validado.\n2. **Inadequação para Grandes Volumes de Dados:** LLMs não são ferramentas de processamento de dados; eles não podem lidar com grandes datasets diretamente. A análise deve ser feita com base em resumos, amostras ou metadados.\n3. **Geração de Código Falho:** O código (Python/R) gerado pode conter erros de sintaxe ou lógica, especialmente se as bibliotecas ou a estrutura de dados não forem descritas com precisão.\n4. **Viés e Simplificação Excessiva:** O LLM pode simplificar demais relações complexas ou introduzir viés nos resultados se o prompt não exigir uma análise multifatorial e crítica.\n5. **Confusão entre Causalidade e Correlação:** O modelo pode interpretar correlação como causalidade em sua interpretação, a menos que o prompt exija cautela e a consideração de variáveis de confusão.
