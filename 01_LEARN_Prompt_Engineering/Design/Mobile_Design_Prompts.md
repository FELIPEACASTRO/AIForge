# Mobile Design Prompts

## Description
**Prompts de Design Móvel** (Mobile Design Prompts) constituem uma subdisciplina da Engenharia de Prompt focada na criação de instruções detalhadas e estruturadas para modelos de Inteligência Artificial, visando a geração, otimização ou análise de elementos de design de interfaces de usuário (UI) e experiência do usuário (UX) especificamente para dispositivos móveis (smartphones e tablets). Esta técnica é essencial para acelerar o ciclo de design, permitindo que designers e desenvolvedores transformem rapidamente ideias conceituais em wireframes, protótipos interativos, ou até mesmo código de interface responsivo. Os prompts devem incorporar as melhores práticas de design móvel, como a priorização de interações *mobile-first*, acessibilidade, design minimalista, e a consideração de gestos e áreas de toque. A eficácia reside na capacidade de especificar o contexto do usuário, o objetivo do aplicativo, a plataforma (iOS/Android), e os componentes visuais e funcionais desejados.

## Examples
```
1. **Geração de Tela:** "Crie o wireframe de uma tela de checkout de e-commerce para iOS. O design deve ser minimalista, com um único botão de 'Pagar Agora' fixo na parte inferior e campos de entrada claros para endereço e método de pagamento." 2. **Otimização de Componente:** "Analise o design atual do nosso menu de navegação inferior (tab bar) para um aplicativo de fitness. Sugira 3 melhorias de UX para aumentar a taxa de cliques no ícone 'Atividade', mantendo a consistência visual com o Material Design." 3. **Design de Interação:** "Gere um prompt para um gerador de micro-interações que crie uma animação sutil de 'curtir' (like) para um aplicativo de rede social. A animação deve ser rápida (máximo 300ms) e usar cores vibrantes." 4. **Acessibilidade:** "Projete uma paleta de cores para um aplicativo de leitura noturna que garanta o contraste AA para usuários com deficiência visual. Forneça os códigos hexadecimais para o fundo, texto primário e elementos de destaque." 5. **Criação de Ícone:** "Desenhe um ícone 3D, moderno e vibrante para um aplicativo de gerenciamento de tarefas. O ícone deve representar 'produtividade' e ser otimizado para a App Store e Google Play (tamanho 1024x1024)." 6. **Fluxo de Usuário:** "Descreva o fluxo de usuário (user flow) ideal para o primeiro login e onboarding de um novo usuário em um aplicativo de meditação. O fluxo deve ter no máximo 4 telas e focar em coletar apenas o nome e a preferência de notificação." 7. **Revisão de Conteúdo:** "Reescreva o texto de erro para uma mensagem de 'Conexão Perdida' em um aplicativo de streaming, tornando-o mais amigável e oferecendo uma solução clara (ex: 'Verificar Configurações de Rede')."
```

## Best Practices
1. **Seja Específico e Estruturado:** Defina claramente o tipo de saída (wireframe, protótipo, código), a plataforma (iOS, Android, ambos), e o estilo visual (Material Design, Human Interface Guidelines, minimalista). 2. **Defina o Contexto do Usuário:** Inclua o público-alvo, o objetivo principal do aplicativo e o cenário de uso para garantir que o design gerado seja contextualmente relevante. 3. **Use Restrições de Design:** Especifique limitações como paleta de cores, tipografia, número máximo de elementos na tela ou requisitos de acessibilidade (WCAG). 4. **Iteração e Refinamento:** Use prompts subsequentes para refinar o resultado inicial, solicitando ajustes em micro-interações, espaçamento ou alinhamento. 5. **Foco em Mobile-First:** Sempre priorize a experiência em telas pequenas, mencionando explicitamente a otimização para toque e a legibilidade em dispositivos móveis.

## Use Cases
1. **Prototipagem Rápida:** Gerar rapidamente múltiplas variações de telas de aplicativos para testes A/B ou validação de conceito. 2. **Geração de Ativos:** Criar ícones, ilustrações e elementos gráficos que se encaixem no estilo visual do aplicativo. 3. **Otimização de UX:** Analisar fluxos de usuário existentes e sugerir melhorias baseadas em princípios de usabilidade móvel. 4. **Criação de Componentes:** Gerar código ou especificações de componentes de UI (botões, formulários, barras de navegação) para bibliotecas de design. 5. **Documentação de Design:** Criar especificações detalhadas de design (cores, tipografia, espaçamento) para desenvolvedores.

## Pitfalls
1. **Prompts Vagos:** Solicitar "um design bonito" sem especificar o público, a função ou o estilo, resultando em saídas genéricas e inutilizáveis. 2. **Ignorar a Plataforma:** Não especificar se o design é para iOS ou Android, levando a inconsistências com as diretrizes nativas (ex: usar um botão flutuante em excesso no iOS). 3. **Foco Apenas na Estética:** Priorizar a aparência visual em detrimento da usabilidade e acessibilidade móvel (ex: fontes muito pequenas, baixo contraste). 4. **Excesso de Complexidade:** Tentar gerar um aplicativo inteiro em um único prompt, em vez de dividir o processo em telas e componentes menores e gerenciáveis. 5. **Desconsiderar o Toque:** Gerar elementos de UI com áreas de toque muito pequenas ou próximas, dificultando a interação do usuário móvel.
