## Papers & Surveys (23)

| Resource | What it is |
|---|---|
| [STAC: When Innocent Tools Form Dangerous Chains to Jailbreak LLM Agents](https://arxiv.org/abs/2509.25624) | Introduces Sequential Tool Attack Chaining (STAC): individually-harmless tool calls that combine across an agent trajectory to enable harmful outcomes only apparent at final execution; >90% ASR on GPT-4.1, with reasoning-driven defenses. Directly on multi-step tool attacks. |
| [ToolFlood: Beyond Selection — Hiding Valid Tools from LLM Agents via Semantic Covering](https://arxiv.org/abs/2603.13950) | Retrieval-layer attack on tool-augmented agents: adversarial tools placed in embedding space dominate retrieval and push benign tools out of context; up to 95% ASR with 1% tool injection on ToolBench. Novel tool-poisoning/availability vector. |
| [Don't Trust Your Upstream: Exploiting LLM Multi-Agent System via Topology-Guided Adversarial Propagation](https://arxiv.org/abs/2512.04129) | Topology-aware multi-hop attack propagating adversarial contamination from exposed edge agents to high-privilege agents in LLM multi-agent systems; 40-78% ASR on standard frameworks, 85% on real apps. Core A2A/multi-agent attack. |
| [GroupGuard: A Framework for Modeling and Defending Collusive Attacks in Multi-Agent Systems](https://arxiv.org/abs/2603.13940) | Models group-collusive attacks where multiple agents coordinate via sociological strategies to mislead a multi-agent system (+15% ASR vs individual); defends via graph monitoring, honeypots, structural pruning (88% detection). Novel multi-agent collusion angle. |
| [Memory Poisoning and Secure Multi-Agent Systems](https://arxiv.org/abs/2603.20357) | Analyzes feasibility of memory poisoning across semantic/episodic/short-term memory architectures in multi-agent systems and proposes cryptographic + private-retrieval defenses; highlights understudied inter-agent memory-integrity risks. |
| [Mind Your HEARTBEAT! Claw Background Execution Inherently Enables Silent Memory Pollution](https://arxiv.org/abs/2603.23064) | Shows heartbeat-driven background agent execution lets untrusted content silently pollute long-term memory (no explicit prompt injection needed); 61% short-term and 76% cross-session behavioral influence. Novel memory-poisoning vector tied to autonomous background loops. |
| [Your LLM Agent Can Leak Your Data: Data Exfiltration via Backdoored Tool Use (Back-Reveal)](https://arxiv.org/abs/2604.05432) | Back-Reveal attack: hidden triggers in fine-tuned agents cause retrieval of stored user info and leakage via disguised tool calls, amplified over repeated multi-turn interactions. Backdoor + tool-use exfiltration. |
| [ClawWorm: Self-Propagating Attacks Across LLM Agent Ecosystems](https://arxiv.org/abs/2603.15727) | First self-replicating worm against a production-scale agent framework (OpenClaw, 40k+ instances): autonomous infection cycles across interconnected LLM agents via multiple backends/infection vectors. Multi-agent propagation threat. |
| [SkillAttack: Automated Red Teaming of Agent Skills through Attack Path Refinement](https://arxiv.org/abs/2604.04989) | Automated red-teaming framework that dynamically verifies exploitability of latent vulnerabilities in non-malicious agent skills from open registries via adversarial prompting; ASR 0.73-0.93 on adversarial, up to 0.26 on real-world skills. Tool/skill supply-chain attack. |
| [How Your Credentials Are Leaked by LLM Agent Skills: An Empirical Study](https://arxiv.org/abs/2604.03070) | Large-scale empirical study (170,226 artifacts, 17,022 sampled skills) finding 520 affected skills with 1,708 security issues; 76.3% require joint NL+code analysis. ASE 2026. Agent-skill credential-leak supply-chain evidence. |
| [Conflict-Aware Retriever Editing for Knowledge Injection Attacks on LLM-Based RAG Systems (CAREATTACK)](https://arxiv.org/abs/2606.18310) | Model-centric RAG poisoning: edits retriever (embedding model) parameters to promote malicious passages into retrieved context rather than injecting synthetic corpus text; targets Qwen3-Embedding, BGE-M3. New retriever-editing attack on agent RAG. |
| [ShieldAgent: Shielding Agents via Verifiable Safety Policy Reasoning](https://arxiv.org/abs/2503.22738) | First guardrail agent that enforces explicit safety-policy compliance over a protected agent's action trajectory using action-based probabilistic rule circuits + formal verification; ships ShieldAgent-Bench (3K instruction/trajectory pairs across 6 web envs, 7 risk categories). Beats prior guards by ~11.3% avg, 90.1% recall, cutting API queries 64.7%. |
| [Securing AI Agents with Information-Flow Control (Fides)](https://arxiv.org/abs/2505.23643) | Microsoft Research formal IFC approach: introduces Fides, a planner using information-flow control with taint-tracking to give provable security guarantees against prompt injection while preserving utility; builds a formal model of enforceable planner properties. |
| [AgentArmor: Enforcing Program Analysis on Agent Runtime Trace to Defend Against Prompt Injection](https://arxiv.org/abs/2508.01249) | Treats agent runtime traces as structured programs: graph constructor builds CFG/DFG/PDG from traces, a property registry attaches tool/data security metadata, and a type system does static inference/checking. Reduces attack success rate to ~3% on AgentDojo with ~1% utility loss. |
| [VeriGuard: Enhancing LLM Agent Safety via Verified Code Generation](https://arxiv.org/abs/2510.05156) | Google (Cloud AI Research / DeepMind) dual-stage framework for formal safety guarantees: offline policy synthesis + formal verification (iterative refinement on verification failures), then a lightweight online runtime monitor validating each action against the pre-verified policy. Targets sensitive domains like healthcare. |
| [DRIFT: Dynamic Rule-Based Defense with Injection Isolation for Securing LLM Agents](https://arxiv.org/abs/2506.12104) | NeurIPS 2025 system-level defense combining a Secure Planner, Dynamic Validator, and Injection Isolator to enforce control- and data-level constraints with dynamically updatable security rules and memory-stream isolation. Cuts AgentDojo ASR from 30.7% to 1.3% with high utility. Code: github.com/SaFo-Lab/DRIFT. |
| [TraceAegis: Securing LLM-Based Agents via Hierarchical and Behavioral Anomaly Detection](https://arxiv.org/abs/2510.11203) | Provenance-based runtime defense that builds a hierarchical abstraction of stable execution units characterizing normal agent behavior, validating traces against hierarchical + behavioral constraints to catch tool poisoning / malicious instructions. Ships TraceAegis-Bench; F1 0.93-0.96, 94.3% red-team anomaly detection. |
| [ISOLATEGPT: An Execution Isolation Architecture for LLM-Based Agentic Systems](https://arxiv.org/abs/2403.04960) | NDSS 2025 execution-isolation architecture using a hub-and-spoke design where third-party app 'spokes' run in isolation, agnostic of each other, with an inter-spoke protocol routing well-defined requests through a hub — containing compromised/ malicious agent apps. A key system-security baseline alongside CaMeL/IsolateGPT. |
| [AgentGuardian: Learning Access Control Policies to Govern AI Agent Behavior](https://arxiv.org/abs/2601.10440) | Learns access-control policies that govern AI-agent behavior, framing agent containment as a learned least-privilege / access-control problem rather than static rules — a 2026 access-control-centric defense complementing MAC frameworks like SEAgent. |
| [CausalArmor: Efficient Indirect Prompt Injection Guardrails via Causal Attribution](https://arxiv.org/abs/2602.07918) | Lightweight IPI guardrail using causal attribution to decide whether a tool call was caused by user intent vs. untrusted observation, aiming for efficiency over heavy counterfactual testing — a 2026 efficient successor in the causal-attribution defense line. |
| [AgentDoG: A Diagnostic Guardrail Framework for AI Agent Safety and Security](https://arxiv.org/abs/2601.18491) | Diagnostic guardrail framework for agent safety/security, benchmarked against leading guard models (LlamaGuard3-8B, LlamaGuard4-12B, Qwen3-Guard, ShieldAgent) — useful as a 2026 comparative guardrail baseline. |
| [VIGIL: Defending LLM Agents Against Tool Stream Injection via Verify-Before-Commit](https://arxiv.org/abs/2601.05755) | 2026 defense introducing a verify-before-commit discipline against 'tool stream injection' — malicious content arriving through streamed tool outputs — validating tool-stream content before the agent commits to acting on it. |
| [TRUSTDESC: Preventing Tool Poisoning in LLM Applications via Trusted Description Generation](https://arxiv.org/abs/2604.07536) | 2026 MCP/tool-poisoning defense that generates trusted tool descriptions so malicious instructions hidden in tool metadata cannot hijack the agent — a generation-side counter to tool/schema poisoning complementing signed-schema and scanner approaches. |
