# Verified Sources (2026 Double-Check) — AI Agent Security

> An expanded, **adversarially-verified** source list produced by a multi-agent double-check (8 discovery angles + WebFetch confirmation). Each entry was confirmed to resolve and be on-topic.

## Benchmarks & Datasets (14)

| Resource | What it is |
|---|---|
| [Agent Security Bench (ASB): Formalizing and Benchmarking Attacks and Defenses in LLM-based Agents](https://arxiv.org/abs/2410.02644) | ICLR 2025 benchmark (code: github.com/agiresearch/ASB) covering 10 scenarios, 400+ tools, and 27 attack/defense methods including 10 prompt-injection attacks, memory poisoning, and a Plan-of-Thought backdoor across 13 LLM backbones. |
| [AgentHarm: A Benchmark for Measuring Harmfulness of LLM Agents](https://arxiv.org/abs/2410.09024) | UK AI Safety Institute / Gray Swan benchmark of malicious multi-step agent tasks to measure refusal robustness and post-jailbreak capability retention. Public dataset on HuggingFace (ai-safety-institute/AgentHarm). |
| [Identifying the Risks of LM Agents with an LM-Emulated Sandbox (ToolEmu)](https://arxiv.org/abs/2309.15817) | ICLR'24 Spotlight: LM-based sandbox that emulates tool execution plus an automatic LM safety evaluator; curated benchmark of 36 high-stakes tools and 144 test cases. Code: github.com/ryoungj/ToolEmu. |
| [SafeAgentBench: A Benchmark for Safe Task Planning of Embodied LLM Agents](https://arxiv.org/abs/2412.13178) | Benchmark for safety-aware task planning of embodied LLM agents in interactive simulation (SafeAgentEnv), covering explicit and implicit hazards with execution + semantic evaluation. |
| [Refusal-Trained LLMs Are Easily Jailbroken As Browser Agents (BrowserART)](https://arxiv.org/abs/2410.13886) | Scale AI red-teaming test suite (BrowserART) of 100 harmful browser-agent behaviors on synthetic and real sites; shows chat-context refusal does not generalize to browser agents. Dataset: huggingface.co/datasets/ScaleAI/BrowserART. |
| [tau-bench: A Benchmark for Tool-Agent-User Interaction in Real-World Domains](https://arxiv.org/abs/2406.12045) | Sierra Research benchmark emulating user-agent conversations under domain-specific API tools and policy documents; relevant to agent safety via policy-compliance/rule-following and the pass^k reliability metric. Code: github.com/sierra-research/tau-bench (tau2-bench successor). |
| [MCPTox: A Benchmark for Tool Poisoning Attack on Real-World MCP Servers](https://arxiv.org/abs/2508.14925) | AAAI-published first benchmark for tool poisoning (malicious instructions in tool metadata) over real Model Context Protocol servers; 45 live servers, 353 tools, 1312 malicious test cases, 20 LLM agents. |
| [Agent-Fence: Mapping Security Vulnerabilities Across Deep Research Agents](https://arxiv.org/abs/2602.07652) | AgentFence security-evaluation framework spanning 14 attack classes across planning, memory, retrieval, and tool-use; tests eight agent architectures and prioritizes real-world failures (unauthorized tool use, wrong-principal actions, authorization confusion) over text safety metrics. |
| [WASP: Benchmarking Web Agent Security Against Prompt Injection Attacks](https://arxiv.org/abs/2504.18575) | A publicly available benchmark (Meta/FAIR) with realistic web-agent hijacking objectives in an isolated environment, used to red-team agentic systems built on VisualWebArena, Claude Computer Use, and Operator. |
| [ST-WebAgentBench: A Benchmark for Evaluating Safety and Trustworthiness in Web Agents](https://arxiv.org/abs/2410.06703) | Enterprise-focused safety/trust benchmark extending WebArena, scoring agents on six orthogonal policy dimensions (e.g., user consent, robustness) with the 'Completion under Policy' (CuP) metric and Risk Ratio. |
| [VPI-Bench: Visual Prompt Injection Attacks for Computer-Use Agents](https://arxiv.org/abs/2506.02456) | Benchmark of 306 interactive test cases across five platforms for visual prompt injection (VPI), where malicious instructions are visually embedded within rendered UIs, targeting computer-use and browser-use agents. |
| [RedTeamCUA: Realistic Adversarial Testing of Computer-Use Agents in Hybrid Web-OS Environments](https://arxiv.org/abs/2505.21936) | Adversarial testing framework with a hybrid VM-OS + Docker-web sandbox and RTC-Bench (864 examples) for indirect prompt injection against frontier CUAs including Operator and Claude Sonnet CUA. ICLR 2026 Oral. |
| [VisualWebArena: Evaluating Multimodal Agents on Realistic Visual Web Tasks](https://arxiv.org/abs/2401.13649) | Foundational benchmark of 910 realistic, visually grounded web tasks on self-hosted environments for multimodal agents; widely reused as the substrate for adversarial/prompt-injection studies (e.g., WASP). |
| [AgentDojo: A Dynamic Environment to Evaluate Prompt Injection Attacks and Defenses for LLM Agents](https://arxiv.org/abs/2406.13352) | NeurIPS 2024 benchmark (Debenedetti, Zhang, Balunovic, Beurer-Kellner, Fischer, Tramer) with 97 realistic tool-use tasks and 629 security test cases for indirect prompt injection; the de facto standard for measuring tool-using agent attack success vs. defenses. Foundational reference for the coding-agent threat model. |

## Research Papers (31)

| Resource | What it is |
|---|---|
| [AdvAgent: Controllable Blackbox Red-teaming on Web Agents](https://arxiv.org/abs/2410.17401) | Black-box red-teaming framework (formerly titled AdvWeb) that injects invisible adversarial prompts into web pages to misdirect VLM/foundation-model web agents; uses a two-stage RL/DPO training paradigm. |
| [AgentPoison: Red-teaming LLM Agents via Poisoning Memory or Knowledge Bases](https://arxiv.org/abs/2407.12784) | NeurIPS 2024 backdoor/memory-poisoning attack targeting RAG-based and memory-based LLM agents (autonomous driving, QA, EHRAgent) via optimized retrieval triggers, no model fine-tuning. |
| [AdvWeb / AdvAgent project & code (web-agent black-box attack)](https://arxiv.org/html/2410.17401v4) | HTML full-text version of the AdvAgent paper (note v2 was titled 'AdvWeb: Controllable Black-box Attacks on VLM-powered Web Agents') — confirms the title evolution and invisible-HTML-injection method details. |
| [AttriGuard: Defeating Indirect Prompt Injection in LLM Agents via Causal Attribution of Tool Invocations](https://arxiv.org/abs/2603.10749) | Runtime defense that treats indirect prompt injection in multi-step tool-calling agents as an action-level causal attribution problem (is this tool call caused by user intent or by untrusted observations?), using parallel counterfactual tests and hierarchical control attenuation. USENIX Security 2026. |
| [Your Agent is More Brittle Than You Think: Uncovering Indirect Injection Vulnerabilities in Agentic LLMs](https://arxiv.org/abs/2604.03870) | Empirical study testing six defenses across nine LLMs in realistic multi-step tool-calling scenarios, showing most defenses fail and that agents execute malicious commands despite high internal uncertainty; proposes representation-engineering detection on hidden states. |
| [ICON: Indirect Prompt Injection Defense for Agents based on Inference-Time Correction](https://arxiv.org/abs/2602.20708) | Two-stage IPI defense: latent-space detection plus attention-manipulation mitigation at inference time, rather than restrictive input filtering; evaluated on OOD and multi-modal agents. |
| [Causality Laundering: Denial-Feedback Leakage in Tool-Calling LLM Agents](https://arxiv.org/abs/2604.04035) | Identifies a multi-step exfiltration class where an attacker probes protected operations, learns from denial responses, and leaks the inferred info through later benign-looking tool calls; proposes ARM (Agentic Reference Monitor) with provenance graphs and counterfactual denied-action edges. |
| [Automating Agent Hijacking via Structural Template Injection](https://arxiv.org/abs/2602.16958) | Phantom: an automated attack framework that injects optimized structured templates into retrieved context to induce role confusion in tool-using agents, using a Template Autoencoder + Bayesian optimization; reports 70+ real-world vulnerabilities. |
| [Memory Poisoning Attack and Defense on Memory Based LLM-Agents](https://arxiv.org/abs/2601.05504) | Empirical memory-poisoning study on persistent-memory agents (clinical data), proposing two defenses: I/O moderation via trust scoring and memory sanitization with temporal filtering; shows realistic legitimate-memory contexts cut attack success vs idealized settings. |
| [Taming Various Privilege Escalation in LLM-Based Agent Systems: A Mandatory Access Control Framework](https://arxiv.org/abs/2601.11893) | Defines agent privilege escalation as actions exceeding least privilege for the user's intended task; introduces SEAgent, an attribute-based mandatory access control framework that monitors agent-tool interactions and defends against confused-deputy variants in multi-agent settings. |
| [Model Context Protocol Threat Modeling and Analyzing Vulnerabilities to Prompt Injection with Tool Poisoning](https://arxiv.org/abs/2603.22489) | STRIDE/DREAD threat model of MCP across five components, identifying client-side tool poisoning (malicious instructions in tool metadata) as the top vulnerability; evaluates seven MCP clients and proposes layered defenses. |
| [MCP-DPT: A Defense-Placement Taxonomy and Coverage Analysis for Model Context Protocol Security](https://arxiv.org/abs/2604.07551) | Layer-aligned taxonomy organizing MCP attacks by the architectural component responsible for enforcement across six MCP layers, with a coverage analysis revealing defense gaps at host orchestration, transport, and supply-chain layers. |
| [ShieldNet: Network-Level Guardrails against Emerging Supply-Chain Injections in Agentic Systems](https://arxiv.org/abs/2604.04426) | Introduces SC-Inject-Bench (10,000+ malicious MCP tools, 25+ attack types) and ShieldNet, a MITM proxy that monitors network interactions to detect MCP supply-chain poisoning at high F1 with low false positives. |
| [The Landscape of Prompt Injection Threats in LLM Agents: From Taxonomy to Analysis](https://arxiv.org/abs/2602.10453) | Survey/taxonomy categorizing prompt-injection attacks by payload-generation strategy and defenses by intervention stage (text/model/execution); introduces the AgentPI benchmark and shows no single defense achieves trustworthiness, utility, and low latency together for context-dependent tool use. |
| [The Attack and Defense Landscape of Agentic AI: A Comprehensive Survey](https://arxiv.org/abs/2603.11088) | Systematic survey of AI-agent security covering design space, attack landscape, and defense mechanisms for agents that couple LLMs with non-AI system components; identifies open gaps. USENIX Security 2026. |
| [Defeating Prompt Injections by Design (CaMeL)](https://arxiv.org/abs/2503.18813) | Google Research / DeepMind / ETH Zurich paper (Debenedetti, Shumailov, Carlini, Tramèr et al.) introducing CaMeL — a capability- and information-flow-control layer that separates control flow from untrusted data; solves 77% of AgentDojo tasks with provable security. |
| [Lessons from Defending Gemini Against Indirect Prompt Injections](https://arxiv.org/abs/2505.14534) | Google DeepMind white paper (Shi, Lin, Hayes, Shumailov et al.) on continuous adversarial evaluation and red-teaming to harden Gemini against indirect prompt injection when the model uses tools on untrusted data. |
| [Red-Teaming LLM Multi-Agent Systems via Communication Attacks (Agent-in-the-Middle / AiTM)](https://arxiv.org/abs/2502.14847) | Introduces Agent-in-the-Middle (AiTM), an attack that compromises an entire LLM multi-agent system by intercepting and manipulating inter-agent messages rather than compromising individual agents; uses an LLM-powered adversarial agent with a reflection mechanism. Accepted at ACL 2025 Findings. |
| [CORBA: Contagious Recursive Blocking Attacks on Multi-Agent Systems Based on Large Language Models](https://arxiv.org/abs/2502.14529) | A denial-of-service-style attack on LLM multi-agent systems that is contagious (propagates across arbitrary network topologies) and recursive (sustained depletion of compute), inducing infinite-loop reasoning / junk-data generation. Evaluated on AutoGen and Camel. |
| [Prompt Infection: LLM-to-LLM Prompt Injection within Multi-Agent Systems](https://arxiv.org/abs/2410.07283) | Demonstrates a self-replicating prompt injection that spreads across interconnected agents like a computer virus, enabling data theft, scams, misinformation, and system-wide disruption; proposes 'LLM Tagging' as a defense. Foundational reference for orchestration-level propagation attacks. |
| [A Survey of LLM-Driven AI Agent Communication: Protocols, Security Risks, and Defense Countermeasures](https://arxiv.org/abs/2506.19676) | 48-page survey formalizing agent communication into a three-layer architecture, analyzing protocols (MCP, A2A) as case studies, dissecting per-layer vulnerabilities, and cataloging defense countermeasures with empirical validation on MCP/A2A. |
| [Open Challenges in Multi-Agent Security: Towards Secure Systems of Interacting AI Agents](https://arxiv.org/abs/2505.02077) | Position paper establishing 'multi-agent security (MASEC)' as a field. Provides an expanded attacker model and threat taxonomy, covering covert collusion, coordinated/cascading attacks, secure protocols, data provenance, and a three-layer governance framework. Authors from Oxford, CMU, MIT, Alan Turing Institute. |
| [Security Threat Modeling for Emerging AI-Agent Protocols: A Comparative Analysis of MCP, A2A, Agora, and ANP](https://arxiv.org/abs/2602.11327) | Comparative threat modeling of four agent protocols (MCP, A2A, Agora, ANP) with an architecture-based risk catalog grounded in trust boundaries, identity/authorization bindings, and cross-protocol composition; lifecycle-aware framework identifying twelve protocol-level risks plus an empirical 'wrong-provider tool execution' case study. |
| [Proposal for Improving Google A2A Protocol: Safeguarding Sensitive Data in Multi-Agent Systems](https://arxiv.org/abs/2505.12490) | Identifies six vulnerabilities in A2A's handling of sensitive data (payment/personal info) and proposes seven concrete improvements: short-lived tokens, Strong Customer Authentication, granular token scopes, explicit consent, intermediary-bypass data transfer, multi-transaction approval, and PSD2 compliance. |
| [Security Analysis of Agentic AI Communication Protocols: A Comparative Evaluation](https://arxiv.org/abs/2511.03841) | First empirical security comparison of agent communication systems using a 14-point vulnerability taxonomy across authentication, authorization, integrity, confidentiality, and availability; finds e.g. CORAL has design strength but implementation flaws and ACP's optional security creates critical gaps. |
| [LlamaFirewall: An open source guardrail system for building secure AI agents (arXiv:2505.03574)](https://arxiv.org/abs/2505.03574) | Peer-style research paper (Meta) describing the LlamaFirewall architecture and its three guardrails for defending agents against prompt injection, agent misalignment, and insecure code. |
| [EIA: Environmental Injection Attack on Generalist Web Agents for Privacy Leakage](https://arxiv.org/abs/2409.11295) | First study of privacy risks of generalist web agents in adversarial environments; proposes Environmental Injection Attack (EIA) injecting malicious web elements into benign HTML to exfiltrate user PII. |
| [EVA: Evolving Semantic Adversaries for Red-Teaming GUI Agents Against Environmental Injection Attacks](https://arxiv.org/abs/2505.14289) | Evolutionary red-teaming framework that generates semantic (not visual) adversarial payloads against MLLM-powered GUI agents, discovering reusable linguistic attack rules. |
| [Your AI, My Shell: Demystifying Prompt Injection Attacks on Agentic AI Coding Editors](https://arxiv.org/abs/2509.22040) | Empirical study (Liu, Zhao, Lyu, Zhang, Wang, Lo) introducing AIShellJack, a framework with 314 attack payloads mapped to 70 MITRE ATT&CK techniques, evaluating prompt injection against GitHub Copilot and Cursor via poisoned external development resources. |
| [Meta SecAlign: A Secure Foundation LLM Against Prompt Injection Attacks](https://arxiv.org/abs/2507.02735) | Defense paper (Chen, Zharmagambetov, Wagner, Guo) presenting the first fully open-source LLM with built-in model-level prompt-injection defense at commercial-grade utility, evaluated on 9 utility and 7 security benchmarks with public weights/code. |
| [Prompt Injection Attacks on Agentic Coding Assistants: A Systematic Analysis of Vulnerabilities in Skills, Tools, and Protocol Ecosystems](https://arxiv.org/abs/2601.17548) | Survey/SoK (Maloyan, Namiot) building a three-dimensional taxonomy of prompt-injection attacks on coding assistants (Claude Code, GitHub Copilot) across skills, tools, and protocol ecosystems, synthesizing dozens of studies and cataloging attack techniques vs. defense effectiveness. Most on-topic single source for the coding-agent angle. |
