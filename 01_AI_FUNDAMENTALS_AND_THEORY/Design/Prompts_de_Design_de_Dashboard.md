# Prompts de Design de Dashboard

## Description
Prompts de Design de Dashboard são instruções detalhadas fornecidas a modelos de IA (como geradores de UI/UX ou LLMs) para criar layouts, wireframes ou especificações de design para painéis de dados (dashboards). O objetivo é acelerar a fase de planejamento e prototipagem, garantindo que o design seja estratégico e focado no usuário. A técnica se baseia em fornecer um "micro-brief" estruturado que detalha o propósito do aplicativo, o tipo de tela, os recursos-chave, o tom visual e, opcionalmente, o objetivo emocional do dashboard. Isso permite que a IA gere rapidamente protótipos de alta fidelidade e relevância funcional.

## Examples
```
**1. Dashboard de Vendas E-commerce (Web):** "Para um sistema de e-commerce B2C chamado 'VendaCerta', projete um dashboard web de alto nível para o Gerente de Vendas. O dashboard deve priorizar a visualização de **KPIs de desempenho diário**, incluindo: **Receita Total**, **Taxa de Conversão**, **Valor Médio do Pedido (AOV)** e um **gráfico de funil de vendas** (Visitas -> Carrinho -> Compra). O layout deve ser **modular**, com os KPIs principais em destaque no topo (cartões grandes) e o funil de vendas em uma seção central. Use uma paleta de cores **corporativa e confiável** (azul e verde) e um tom visual **focado em resultados e eficiência**."

**2. Dashboard de Monitoramento de Pacientes (Tablet):** "Para um aplicativo de saúde hospitalar chamado 'CuraRápida', crie um dashboard otimizado para tablet para a equipe de Enfermagem. O dashboard deve mostrar o **status de monitoramento de 5 pacientes críticos** simultaneamente. Para cada paciente, exiba: **Frequência Cardíaca (em tempo real)**, **Saturação de Oxigênio**, **Próxima Dose de Medicação** e um **alerta visual** claro para desvios de parâmetros vitais. O layout deve ser em **grade**, com cartões de paciente que mudam de cor (verde/amarelo/vermelho) conforme a criticidade. O tom visual deve ser **clínico, limpo e de alta urgência/clareza**."

**3. Dashboard de Análise de Mídias Sociais (Web):** "Para uma ferramenta de análise de marketing digital chamada 'Viralize', gere um dashboard web para o Analista de Mídias Sociais. O foco deve ser a **comparação de desempenho entre 3 plataformas** (Instagram, TikTok, LinkedIn). Inclua: **Crescimento de Seguidores (gráfico de linha)**, **Taxa de Engajamento por Plataforma (gráfico de barras)** e uma **nuvem de palavras-chave** dos comentários mais recentes. O layout deve ser **vertical e informativo**, com seções bem definidas para cada métrica. Use cores **vibrantes e modernas** e um tom visual **analítico e orientado a tendências**."

**4. Dashboard de Finanças Pessoais (Mobile):** "Para um aplicativo de gestão financeira pessoal chamado 'BolsaLeve', projete um dashboard móvel para o usuário final. O objetivo é promover o **controle orçamentário e a motivação para economizar**. Exiba: **Saldo Atual (em destaque)**, **Gráfico de Pizza de Distribuição de Gastos (por categoria)** e um **medidor de progresso** em anel para a meta de economia mensal. O layout deve ser **minimalista e amigável**, com navegação fácil para adicionar transações. Use uma paleta de cores **suave e motivacional** (tons de amarelo e cinza claro) e um tom visual **de apoio e não-julgamento**."

**5. Dashboard de Energia Residencial IoT (Mobile):** "Para um sistema de casa inteligente chamado 'EcoLar', crie um dashboard móvel para o proprietário da casa. O dashboard deve focar na **eficiência energética e na economia de custos**. Inclua: **Consumo de Energia em Tempo Real (medidor)**, **Comparativo de Consumo Semanal (gráfico de área)** e um **cartão de 'Economia Sugerida'** com base em padrões de uso. O layout deve ser **escuro (dark mode)** para economizar bateria e destacar os dados. Use cores **tecnológicas e frias** (azul elétrico e ciano) e um tom visual **de controle e inteligência**."

**6. Dashboard de Performance de Suporte ao Cliente (Web):** "Para um software de Help Desk chamado 'AtendeBem', projete um dashboard web para o Supervisor de Suporte. O foco é a **qualidade e velocidade do serviço**. Exiba: **Tempo Médio de Resposta (TMR)**, **Taxa de Resolução no Primeiro Contato (FCR)** e um **gráfico de dispersão** mostrando a satisfação do cliente (CSAT) versus o tempo de resolução. O layout deve ser **denso, mas organizado**, com filtros de data proeminentes. Use cores **neutras com alertas em vermelho/verde** para métricas fora da meta. O tom visual deve ser **objetivo e de gestão de desempenho**."

**7. Dashboard de Progresso Educacional (Web):** "Para uma plataforma de e-learning chamada 'SaberMais', crie um dashboard web para o aluno. O objetivo é **manter o aluno engajado e ciente de seu progresso**. Inclua: **Porcentagem de Conclusão do Curso (medidor de progresso)**, **Gráfico de Barras de 'Tempo Dedicado por Semana'** e uma **seção de 'Próximos Passos'** com o módulo a ser iniciado. O layout deve ser **limpo e encorajador**, com elementos de gamificação (badges ou pontos). Use cores **claras e estimulantes** (laranja e roxo) e um tom visual **de incentivo e conquista**."
```

## Best Practices
**Seja um Estrategista:** Pense no objetivo de negócio e no usuário final antes de escrever o prompt. **Seja Específico:** Inclua sugestões de estrutura de UI (ex: layout em cartão, rastreador em anel) e elementos específicos (ex: barra de hidratação, seletor de humor). **Use Cues de Tom Emocional:** Palavras como "calmante", "divertido", "minimalista" ajudam a IA a definir o estilo. **Nomeie o Aplicativo:** Mesmo que seja fictício, dar um nome ajuda a IA a criar uma identidade visual coesa. **Pense como um Product Owner:** O prompt deve ser um resumo do que o produto precisa fazer.

## Use Cases
**Prototipagem Rápida:** Gerar wireframes iniciais em segundos para validação de conceito. **Exploração de Layout:** Testar diferentes arranjos de elementos (modular, em coluna, em grade) para otimizar a densidade de informação. **Design de BI (Business Intelligence):** Criar layouts para ferramentas como Tableau ou Power BI, focando na hierarquia de métricas. **Dashboards de Aplicativos Móveis:** Projetar telas de resumo de atividades, saúde ou finanças pessoais. **Re-design:** Gerar novas ideias de layout para dashboards existentes que precisam de uma atualização de UX.

## Pitfalls
**Prompts Genéricos:** Pedir apenas "Design um dashboard" resulta em layouts sem alma, estratégia ou fluxo. **Foco Apenas na Estética:** Ignorar a **função** e as **métricas** chave, resultando em um dashboard bonito, mas inútil. **Excesso de Informação:** Tentar incluir métricas demais, o que leva a um design poluído e confuso. O prompt deve focar nas **prioridades**. **Falta de Contexto:** Não especificar o público-alvo ou o propósito do dashboard.
