# Mixture Models and EM

> Latent-variable density models that represent data as a weighted blend of simpler component distributions, fit by the Expectation-Maximization (EM) algorithm.

## Why it matters

Mixture models are the canonical tool for **soft clustering, density estimation, and unsupervised structure discovery** when data is generated by several unobserved subpopulations. The **EM algorithm** that fits them is a general-purpose recipe for maximum-likelihood estimation under missing/latent data, underpinning hidden Markov models, factor analyzers, topic models, and missing-data imputation. Together they give a principled, probabilistic alternative to hard methods like k-means, with calibrated cluster memberships and a likelihood you can use for **model selection**.

## Core concepts

**Finite mixture.** A density written as a convex combination of $K$ components:

$$p(x \mid \theta) = \sum_{k=1}^{K} \pi_k \, p(x \mid \phi_k), \qquad \pi_k \ge 0,\ \textstyle\sum_k \pi_k = 1.$$

Each $\pi_k$ is a **mixing weight**; $\phi_k$ are the parameters of component $k$. Introduce a latent indicator $z \in \{1,\dots,K\}$ with $p(z=k)=\pi_k$ and $p(x \mid z=k)=p(x \mid \phi_k)$; marginalizing $z$ recovers the mixture.

**Gaussian Mixture Model (GMM).** The components are Gaussians, $p(x \mid \phi_k)=\mathcal{N}(x \mid \mu_k, \Sigma_k)$. The covariance structure (`full`, `tied`, `diag`, `spherical`) controls capacity vs. parameter count and is itself a model-selection choice.

**Responsibilities.** The posterior membership of point $x_i$ to component $k$:

$$\gamma_{ik} = p(z_i = k \mid x_i) = \frac{\pi_k\, \mathcal{N}(x_i \mid \mu_k,\Sigma_k)}{\sum_{j}\pi_j\, \mathcal{N}(x_i \mid \mu_j,\Sigma_j)}.$$

**EM algorithm.** Direct maximization of the log-likelihood $\sum_i \log \sum_k \pi_k p(x_i\mid\phi_k)$ is hard (log-of-sum). EM alternates:

- **E-step:** compute responsibilities $\gamma_{ik}$ using current parameters.
- **M-step:** re-estimate parameters as responsibility-weighted statistics, e.g. for a GMM
  $N_k=\sum_i \gamma_{ik}$, $\ \mu_k = \frac1{N_k}\sum_i \gamma_{ik} x_i$, $\ \Sigma_k = \frac1{N_k}\sum_i \gamma_{ik}(x_i-\mu_k)(x_i-\mu_k)^\top$, $\ \pi_k = N_k / N$.

EM maximizes a lower bound (the **ELBO / free energy** $\mathcal{F}(q,\theta) = \mathbb{E}_q[\log p(x,z\mid\theta)] + H[q]$). It **monotonically increases the likelihood** and converges to a stationary point — typically a local optimum, so multiple random restarts and good initialization (often via k-means) matter.

**Pitfalls.** Likelihood is non-convex and unbounded for unconstrained GMMs (a component can collapse onto a single point, $\Sigma_k \to 0$); use covariance regularization / a prior. Label switching makes components non-identifiable. Choosing $K$ requires explicit model selection.

**Model selection (AIC/BIC).** Pick $K$ (and covariance type) by penalizing fit with complexity. With log-likelihood $\hat{\mathcal{L}}$, $p$ free parameters, $n$ samples:

$$\text{AIC} = 2p - 2\hat{\mathcal{L}}, \qquad \text{BIC} = p\ln n - 2\hat{\mathcal{L}}.$$

Lower is better. **BIC** penalizes complexity more heavily and is consistent for recovering the true model; **AIC** targets predictive performance and tends to pick larger $K$. Alternatives: held-out likelihood / cross-validation, and **Dirichlet-Process / variational Bayesian GMMs** that infer the effective number of components automatically.

## Variants

| Method | Idea | When to use |
|---|---|---|
| **Gaussian Mixture Model (GMM)** | Mixture of Gaussians, fit by EM | Continuous data, soft clustering, density estimation |
| **k-means** | Hard-assignment limit of a spherical, equal-weight GMM | Fast baseline; well-separated spherical clusters |
| **Bernoulli / categorical mixture** | Components are discrete distributions | Binary or categorical features (e.g. document/term data) |
| **Mixture of factor analyzers (MFA)** | Each component is a low-rank Gaussian | High-dimensional data, dimensionality-aware clustering |
| **Variational Bayesian GMM** | Mean-field VI with Dirichlet prior on weights | Automatically prunes unused components; uncertainty over params |
| **Dirichlet Process GMM (infinite mixture)** | Nonparametric prior; $K$ grows with data | Unknown / open-ended number of clusters |
| **Hidden Markov Model (HMM)** | Mixture with Markov-dependent latent states | Sequential / temporal data |
| **Stochastic / online EM** | Mini-batch M-steps | Large or streaming datasets |
| **Classification EM (CEM)** | Hard E-step (assign argmax) | Faster, sharper clusters; relates to k-means |
