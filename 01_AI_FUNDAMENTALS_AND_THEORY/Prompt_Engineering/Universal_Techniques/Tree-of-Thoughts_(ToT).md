# Tree-of-Thoughts (ToT)

## Description
**Tree-of-Thoughts (ToT)** is a framework that generalizes the **Chain-of-Thought (CoT)** prompting technique. Instead of a linear sequence of reasoning steps, ToT structures the problem-solving process as a **tree**, where 'thoughts' are coherent, intermediate units of text. This allows the Large Language Model (LLM) to explore multiple reasoning paths simultaneously, perform self-evaluation of choices, look ahead, and backtrack when a path proves unproductive. This approach enables more deliberate, 'System 2'-type reasoning, significantly improving the LLM's ability to handle complex tasks that require non-trivial planning, search, or strategic decision-making.

## Examples
```
**Example 1: Game of 24 (Logic)**\n**Objective:** Use the numbers 4, 6, 7, 8 exactly once with standard arithmetic operations (+, -, *, /) to reach 24.\n**Prompt:**\n1. **Generate Thoughts:** Propose 3 distinct initial combinations (for example, two combined numbers) and the remaining numbers for each path.\n2. **Evaluate Thoughts:** Score each path (1-5) based on how close the result is to a factor of 24 (for example, 2, 3, 4, 6, 8, 12) or to 24 itself.\n3. **Select and Continue:** Choose the path with the highest score and repeat the process (Generate, Evaluate, Select) until 24 is reached or all paths are exhausted.\n4. **Final Answer:** State the final expression.\n\n**Example 2: Creative Writing (Plot Planning)**\n**Objective:** Develop the next major plot point for a fantasy novel where the protagonist, a young mage, has just discovered a powerful, hidden artifact.\n**Prompt:**\n1. **Generate Thoughts:** Create 3 distinct, high-stakes plot options for the immediate effect of the artifact on the world (for example, 'It unleashes a magical storm', 'It awakens an ancient guardian', 'It is immediately stolen by the antagonist').\n2. **Evaluate Thoughts:** Evaluate each option based on (a) dramatic tension, (b) alignment with the character arc, and (c) originality (1-5).\n3. **Select and Continue:** Select the option with the highest combined score. Then, generate 3 sub-thoughts (next steps) for the chosen plot point.\n4. **Final Answer:** Describe the chosen plot point and its immediate consequences in detail.\n\n**Example 3: Strategic Planning (Business Decision)**\n**Objective:** Determine the best market entry strategy for a new sustainable coffee brand in a saturated market.\n**Prompt:**\n1. **Generate Thoughts:** Propose 3 initial market entry strategies (for example, 'Focus on e-commerce and social media marketing', 'Partner with a large supermarket chain', 'Open a flagship store in a high-traffic area').\n2. **Evaluate Thoughts:** For each strategy, assess the (a) initial cost, (b) potential reach, and (c) risk level (Low, Medium, High).\n3. **Select and Continue:** Choose the strategy with the best balance between reach and risk. Then, generate 3 implementation sub-thoughts (for example, 'Define the first 3 marketing actions', 'Identify 3 potential partners', 'Select 3 target locations').\n4. **Final Answer:** Present the chosen strategy, its rationale, and the first three steps of the implementation plan.\n\n**Example 4: Code Debugging (Root Cause Analysis)**\n**Objective:** A Python function that processes user data is failing intermittently with a 'KeyError'. Analyze the potential root causes.\n**Prompt:**\n1. **Generate Thoughts:** Propose 3 distinct categories of potential root causes (for example, 'Input Data Validation Failure', 'Asynchronous Race Condition', 'External API Change').\n2. **Evaluate Thoughts:** For each category, propose a specific test or log to confirm the hypothesis. Score the likelihood of each cause (1-5).\n3. **Select and Continue:** Select the most likely cause. Generate 3 specific sub-thoughts (for example, 'Check the data schema validation code', 'Review the API documentation for recent changes', 'Add a locking mechanism').\n4. **Final Answer:** Identify the most likely root cause and the recommended fix.\n\n**Example 5: Crosswords (Intensive Search)**\n**Objective:** Solve a 3x3 crossword puzzle with the following clues: Across 1: 'Fast vehicle' (3 letters), Down 1: 'Small insect' (3 letters), Down 2: 'A type of tree' (3 letters).\n**Prompt:**\n1. **Generate Thoughts:** For the first clue ('Fast vehicle'), generate 3 possible 3-letter words (for example, 'CAR', 'JET', 'BUS').\n2. **Evaluate Thoughts:** For each word, check whether it allows valid words for the intersecting down clues. Score based on the number of valid intersection possibilities.\n3. **Select and Continue:** Choose the word with the highest score. Repeat the process for the next intersecting clue, generating new thoughts and evaluating them based on the remaining constraints.\n4. **Final Answer:** Provide the filled 3x3 grid.
```

## Best Practices
**Define the Thought Structure:** Clearly instruct the model on how to generate the 'thoughts' (for example, 'Generate 3 distinct approaches to solve this problem.').\n**Implement Self-Evaluation:** Require the model to evaluate the quality or potential of each generated thought/path (for example, 'Score each approach from 1 to 5 based on feasibility and likelihood of success.').\n**Use a Search Strategy:** Specify the search mechanism (for example, Breadth-First Search (BFS) to explore many options, or Depth-First Search (DFS) for a deeper exploration of a single path).\n**Iterative Refinement:** Instruct the model to select the best thought and continue the process, or to backtrack if the current path fails.\n**Keep Thoughts Concise:** Thoughts should be coherent but not overly verbose, in order to manage the context window size.

## Use Cases
**Complex Mathematical and Logic Puzzles** (for example, Game of 24).\n**Creative Writing and Story Planning** (generating multiple plot points and selecting the best one).\n**Strategic Planning and Decision Support** (evaluating different business or military strategies).\n**Code Generation and Debugging** (exploring multiple implementation or fix strategies).\n**Crosswords and Other Search-Intensive Tasks**.

## Pitfalls
**High Computational Cost:** ToT requires significantly more tokens and processing time than CoT due to the exploration of multiple branches.\n**Context Window Overflow:** Generating too many thoughts or very long thoughts can quickly exceed the LLM's context window, especially for deep searches.\n**Suboptimal Evaluation:** The model's self-evaluation mechanism may be flawed, leading it to prune a promising path or to follow a dead end.\n**Over-Specification:** Overly rigid instructions about the tree structure can limit the model's natural reasoning flexibility.\n**Implementation Complexity:** Requires more complex prompting or external code (for example, Python) to manage the tree structure and the search process.
