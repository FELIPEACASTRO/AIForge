# Code Refactoring Prompts

## Description
Code refactoring with the use of Artificial Intelligence (AI) is the process of restructuring existing source code to improve its internal structure, design, and readability, without changing its external behavior. The main goal is to increase the maintainability, extensibility, and efficiency of the code. Generative AI, such as ChatGPT, acts as a coding assistant, allowing developers to use natural-language prompts to request specific improvements, such as applying design patterns (e.g., Factory Pattern), eliminating code duplication, improving performance, adhering to clean code principles (e.g., SOLID), and implementing secure coding practices. The use of well-crafted prompts is crucial to guide the AI to produce accurate and contextually relevant refactorings, transforming a process that was time-consuming into a faster and safer task, especially when combined with unit tests for validation. ## Examples
```
**1. Refactoring to a Design Pattern:** "Refactor the following Java code to implement the Factory pattern. The goal is to decouple object creation from usage logic. Keep the original functionality. [Insert Java code here]" **2. Performance Improvement:** "Analyze this Python script and suggest refactorings to improve its performance, focusing on the optimization of loops and data structures. [Insert Python code here]" **3. Duplication Elimination (DRY):** "Identify and eliminate code duplication in this JavaScript module. Create a reusable utility function for the repeated code. [Insert JavaScript code here]" **4. Applying SOLID Principles:** "Refactor this C# class to adhere to the Single Responsibility Principle (SRP) of SOLID. Split the class into smaller components if necessary. [Insert C# code here]" **5. Readability and Idiomaticity Improvement:** "Refactor this JavaScript function to use modern features, such as `async/await` and *arrow functions*, making it more idiomatic and readable. [Insert JavaScript code here]" **6. Security Refactoring:** "Refactor this code snippet to implement secure coding practices, specifically input validation and *output encoding*, to prevent XSS. [Insert code here]" **7. Legacy Code Refactoring:** "This is a legacy code snippet in PHP. Refactor it to use modern PHP 8 syntax, improving clarity and maintainability. [Insert PHP code here]"
```

## Best Practices
**1. Provide Complete Context:** Include the full code snippet to be refactored, the programming language, and the specific goal of the refactoring (e.g., "improve performance", "increase readability", "apply the Observer pattern"). **2. Define Constraints and Principles:** Explicitly ask the AI to adhere to clean code principles (e.g., SOLID, DRY) or specific design patterns. **3. Use Tests as Validation:** Whenever possible, include unit tests or ask the AI to generate tests before and after the refactoring to ensure that the external behavior of the code was not changed. **4. Incremental Refactoring:** For large code bases, request refactorings in smaller, manageable parts, instead of trying to refactor the entire file at once. **5. Ask for Explanations:** Request that the AI explain the changes made and justify the design decisions, which serves as a code review and a learning opportunity.

## Use Cases
**1. Maintainability Improvement:** Restructure complex modules to make them easier to understand and maintain by new team members. **2. Performance Optimization:** Identify and rewrite code bottlenecks (e.g., inefficient algorithms, database queries) to reduce runtime. **3. Applying Patterns:** Convert procedural code into object-oriented code or apply design patterns (e.g., Singleton, Strategy) to increase flexibility and reuse. **4. Testability Improvement:** Refactor code to make it more modular and decoupled, facilitating the writing of unit tests. **5. Legacy Code Modernization:** Update the syntax and practices of old code to the latest versions of the programming language, reducing technical debt. **6. Security Reinforcement:** Insert or improve security mechanisms (e.g., input validation, exception handling) in vulnerable areas of the code.

## Pitfalls
**1. Excessive Trust:** Accepting the AI's refactoring without a human review and without running tests. The AI may introduce subtle errors or break the business logic. **2. Lack of Context:** Providing only a small code snippet without the project context (e.g., dependencies, global variables), leading to incomplete or incorrect refactorings. **3. Vague Prompts:** Using generic prompts like "improve this code" without specifying the goal (performance, readability, security), resulting in changes that do not meet the real need. **4. Ignoring *Code Smells*:** Not asking the AI to identify and resolve specific *code smells* (e.g., long functions, duplication, dependence on global variables), missing the opportunity for a deeper refactoring. **5. Contract Breaking:** The AI's refactoring may inadvertently change function signatures or API behavior, which should always be verified.
